Version 0.6.5 02/17/02
Bugs fixed
  • [#212530] Some commands fail on read-only files

    jVi, when running under JBuilder, now has an option to work around this jdk1.3 issue. The problem is that if JEditorPane.isEditable() is false, then KEY_TYPED events are not delivered to the application. Stuff like '^D' still gets through, but normal asci does not.

    The fix involves changing 'isEditable()' while the window is the currently active editor. This behavior, which defaults on, can be turned off with the 'hack read-only' option in the 'Editor' tab of the 'EditorOptions' dialog.
